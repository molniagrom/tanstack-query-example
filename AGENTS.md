# AGENTS.md

Repository working notes for automated agents and maintainers.

## Project Snapshot

- Vite + React 19 + TypeScript application.
- State/data layer uses TanStack Router and TanStack Query.
- The codebase follows a feature-oriented layout under `src/app`, `src/features`, `src/widgets`, `src/pages`, and `src/shared`.
- Serverless/API helpers live in `api/`.

## Common Commands

- `pnpm dev` starts the local Vite dev server.
- `pnpm build` runs TypeScript checks and production build.
- `pnpm lint` runs ESLint across the repo.
- `pnpm router:generate` regenerates TanStack Router route artifacts.
- `pnpm api:gen` regenerates the OpenAPI client schema in `src/shared/api/schema.ts`.
- `pnpm preview` serves the production build locally.

## Generated Files

- `src/app/routes/routeTree.gen.ts` is generated by TanStack Router.
- `src/shared/api/schema.ts` is generated by `pnpm api:gen`.
- Do not edit generated files by hand unless the task explicitly requires it.
- Re-run generators after changing route definitions or the upstream API schema.

## Editing Rules

- Keep changes small and localized to the relevant feature or layer.
- Follow existing import and folder conventions; prefer the current relative-import style used in the repo.
- Keep TypeScript strict-safe. Avoid `any` unless there is a clear, unavoidable reason.
- Reuse shared query keys, API helpers, and UI primitives instead of duplicating them.
- Preserve existing CSS module and styling patterns.
- Avoid adding dependencies unless they are necessary for the task.

## Verification

- Run `pnpm lint` for code changes.
- Run `pnpm build` when route generation, data fetching, or production behavior changes.
- If a task touches routes, regenerate the route tree before validating the result.

## Environment Notes

- Local environment variables are stored in `.env` and `.env.local`.
- Do not commit secrets or machine-specific values.
